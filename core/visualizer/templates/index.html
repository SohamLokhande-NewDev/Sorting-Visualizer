Foundations

Data Structures

Understanding the building blocks that make efficient sorting possible. Each structure has unique properties that different algorithms exploit.

🧱
Arrays
Contiguous memory blocks. O(1) random access makes them ideal for in-place sorting. Most sorting algorithms operate directly on arrays.
🔗
Linked Lists
Dynamic structure with pointer-based traversal. Insertion and deletion are O(1) but random access is O(n), making merge sort naturally suited.
🌳
Binary Trees
Hierarchical structure. Binary search trees maintain sorted order inherently. Heap sort leverages the heap property for O(n log n) performance.
🗂️
Hash Tables
Key-value storage with O(1) average lookups. Used in counting sort and radix sort variants for non-comparison based linear time sorting.
📚
Stacks & Queues
LIFO and FIFO structures. Stacks drive iterative quicksort. Queues are fundamental to merge sort's merge step.
🎯
Heaps
Complete binary tree with the heap property. Enables O(n log n) heap sort and efficient priority queues for scheduling algorithms.
